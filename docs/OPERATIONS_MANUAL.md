# Kurukuru Monitor 運用マニュアル

## 1. はじめに

Kurukuru Monitor は、車両の位置情報とカメラ映像をリアルタイムで確認し、運行状況を把握するための監視システムです。オペレーターは地図画面と映像画面を使い、車両の現在地、通信状態、監視カメラの状況を日常業務の中で確認します。

本書は、配車担当者、監視担当者、現場オペレーター向けの実務マニュアルです。専門的な設定変更ではなく、日々の監視と一次切り分けに必要な内容に絞って記載します。

## 2. 起動方法

### アプリ起動

1. 端末の電源とネットワーク接続を確認します。
2. Kurukuru Monitor デスクトップアプリを起動します。
3. 地図画面またはメイン画面が表示されることを確認します。
4. システム状態表示に異常がないかを確認します。

起動直後は、API 接続、地図表示、車両データ取得、カメラ接続状況が順次反映されます。数秒待っても情報が出ない場合は、本書の「よくある問題」を参照してください。

## 3. 主な画面

### Map Screen

地図画面では、車両の現在位置や状態を確認できます。

確認できる主な内容:

- 車両位置
- 車両の状態表示
- 地図上の Place Marker
- Follow モード
- 3D モード

状態表示の基本例:

- `Online`
  通常通信中
- `Delayed`
  更新遅延あり
- `Offline`
  更新途絶または通信断

### 車両選択

地図上の車両を選択すると、その車両を中心に確認しやすくなります。選択後に Follow モードを有効にすると、対象車両を追従表示できます。

## 4. Follow Mode

### できること

Follow Mode は、選択した車両を地図上で追いかけて表示しやすくする機能です。移動中の車両を継続監視したい場合に使用します。

### 使い方

1. 地図上で対象車両を選択します。
2. Follow ボタンを押します。
3. Follow が有効になると、対象車両に合わせて地図表示が追従します。

不要になった場合は、Follow ボタンを再度押すか、地図を手動で移動して解除します。

## 5. 3D Mode

### できること

3D Mode は、地図を傾けて立体的に表示する機能です。進行方向や周辺地形を感覚的に把握しやすくなります。

### 使い方

1. 地図画面の `3D` ボタンを押します。
2. ボタンが有効表示になると 3D 表示に切り替わります。
3. もう一度押すと通常の平面表示に戻ります。

必要に応じて回転ボタンや北向きリセットを使用してください。

## 6. Place Markers

Place Marker は、地点情報を地図上で共有するための機能です。

### 作成

1. 地図上の対象位置を選びます。
2. Place Marker 作成操作を行います。
3. 必要な名称やメモを入力して保存します。

### 編集

1. 既存 Marker を選択します。
2. 編集メニューから名称や内容を更新します。

### 削除

1. 対象 Marker を選択します。
2. 削除操作を実行します。
3. 誤削除を防ぐため、確認表示がある場合は内容を確認して確定します。

## 7. Video Wall

### できること

Video Wall では複数カメラの映像を同時表示できます。車両周辺や拠点映像をまとめて監視する用途に適しています。

### 操作方法

1. Video Wall 画面を開きます。
2. 必要なカメラまたはレイアウトを選択します。
3. 各映像が正常に再生されるか確認します。

### 確認ポイント

- 映像が止まっていないか
- 対象カメラが正しいか
- 通信遅延や切断がないか

## 8. System Status

オペレーターは、次の項目を定期的に確認してください。

- API 接続状態
- 車両数が正しく表示されているか
- 地図が正常表示されているか
- カメラ映像が再生されているか
- デスクトップ状態が異常表示になっていないか

異常がある場合は、まずネットワーク、API、対象カメラ、対象車両の順で切り分けます。

## 9. 日常運用手順

### 始業時

1. システムを起動します。
2. 地図が表示されることを確認します。
3. 車両一覧または車両マーカーが表示されることを確認します。
4. カメラ映像が取得できることを確認します。
5. 必要な監視対象を Follow または画面レイアウトに設定します。

### 運用中

1. 車両の移動状況を監視します。
2. 遅延やオフライン表示がないか確認します。
3. 必要に応じて Follow Mode や 3D Mode を使い分けます。
4. カメラ映像に途切れがないか確認します。
5. Place Marker を活用し、監視地点や注意地点を共有します。

### 終業時

1. 異常車両や未確認事項がないか確認します。
2. 必要があれば引継ぎメモを残します。
3. システム終了手順に従ってアプリを閉じます。

## 10. よくある問題

### 車両データが表示されない

- API サーバーが起動しているか確認してください。
- ネットワーク接続を確認してください。
- 対象車両端末が送信中か確認してください。

### 地図が表示されない

- Google Maps または Mapbox の設定不備がないか確認してください。
- API キー未設定や期限切れの可能性を確認してください。

### カメラ映像が表示されない

- MediaMTX が動作しているか確認してください。
- 対象カメラの配信元が稼働しているか確認してください。
- ネットワーク経路に問題がないか確認してください。

### システムが切断状態に見える

- API 接続状態を確認してください。
- 端末ネットワークを確認してください。
- 車両側またはカメラ側の電源・通信状態を確認してください。

## 11. エスカレーション手順

次のように連絡先を使い分けてください。

### システム管理者へ連絡するケース

- アプリが起動しない
- API に接続できない
- 複数端末で同時に同じ障害が出る

### ネットワーク管理者へ連絡するケース

- 社内ネットワークや VPN が不安定
- カメラ配信や API 接続がネットワーク断で失敗している
- 端末からサーバーへの疎通に問題がある

### 開発チームへ連絡するケース

- 車両表示ロジックが明らかに不正
- Follow / 3D / Place Marker など UI 機能が継続的に異常
- 映像や地図の表示がソフトウェア不具合と思われる

エスカレーション時は、発生時刻、対象車両、対象カメラ、画面表示内容、再現手順を整理して共有してください。

## 12. 運用上の注意

- 認証情報や API キーは一般利用者へ共有しないでください。
- 地図や映像が一時的に更新されない場合でも、まずは通信状況を確認してください。
- 障害時は慌てて再設定せず、表示状態とログの変化を確認してから連絡してください。
