# Operation Manual

## 1. システム起動

1. Ubuntu PC を起動します。
2. 自動起動後、`kurukuru-monitor` の操作画面が表示されることを確認します。
3. 自動起動しない場合は、サービス状態を確認します。

```bash
./scripts/status-services.sh
```

## 2. 映像ウォールを開く

1. 操作画面の `映像ウォール / Video Wall` ボタンを押します。
2. 4画面の監視レイアウトが開くことを確認します。
3. カメラが未設定または停止中の場合は、タイル内の案内表示を確認します。

## 3. 地図画面を開く

1. 操作画面の `地図画面 / Map` ボタンを押します。
2. 車両マーカー、色、名称、更新状態を確認します。
3. Mapbox トークン未設定時は簡易地図表示になります。

## 4. カメラを拡大表示する

1. 映像ウォールの対象カメラをダブルクリックします。
2. または `Open Popup` を押します。
3. 大画面プレーヤーが開き、個別の再接続確認ができます。

## 5. システム状態を確認する

1. 操作画面の `システム状態 / System Status` ボタンを押します。
2. 以下を確認します。
   - API 稼働状態
   - カメラ状態
   - GPS 状態
   - 受信機状態
   - データベース状態

## 6. カメラや車両を追加・編集する

1. 操作画面の `設定 / Settings` を開きます。
2. 車両名、色、有効/無効を設定します。
3. カメラ設定では以下を入力します。
   - 車両
   - カメラ名称
   - 種別
   - ベンダー
   - RTSP URL
   - ユーザー名
   - パスワード
4. 必要に応じて `Test Camera` を実行します。
5. パスワード欄を空のまま保存すると既存パスワードが維持されます。

## 7. 現地試験を実行する

1. 操作画面の `現地試験 / Field Test` を開きます。
2. 担当者名を入力して試験を開始します。
3. 各項目を `Pass / Fail / Reset` で記録します。
4. 試験完了後にレポートを出力します。

## 8. バックアップ / 復元

バックアップ:

```bash
./scripts/backup-settings.sh
```

復元:

```bash
./scripts/restore-settings.sh ./runtime/backups/<timestamp>
./scripts/restart-services.sh
```

## 9. トラブル対応

### カメラ映像が出ない

- `mpv` が導入済みか確認
- 設定画面で RTSP URL と認証情報を確認
- システム状態画面でカメラ状態を確認

### 地図が表示されない

- `VITE_MAPBOX_ACCESS_TOKEN` を確認
- デモモードかどうかを確認
- GPS または API 状態を確認

### GPS が更新されない

- SE220 設定と配線を確認
- `/system/status` と API ログを確認
- デモモードであれば模擬ルート表示か確認

### 自動起動しない

- `./scripts/status-services.sh`
- `./scripts/view-logs.sh`
- `docs/ubuntu-deployment.md` の自動起動確認手順を参照
