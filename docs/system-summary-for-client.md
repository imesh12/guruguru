# 車両監視システム概要

## 1. システムの目的

本システムは、市役所業務における車両映像確認および位置確認を分かりやすく行うための監視支援システムです。運用担当者が少ない操作で状況を確認できることを目的としています。

## 2. システム概要

本システムは、以下の機能を1台の受信PC上で提供します。

- 車載カメラ映像の表示
- 車両位置情報の地図表示
- システム状態の監視
- カメラ・車両設定の管理
- 現地試験および受入確認記録

## 3. 主な機能

- 4画面映像ウォール表示
- 車両位置のリアルタイム地図表示
- カメラ状態、GPS状態、受信機状態の見える化
- 設定画面からの車両・カメラ追加編集
- 現地試験レポート出力
- Ubuntu自動起動対応

## 4. 運用前提

- 受信PCはUbuntu環境で常時稼働します。
- カメラ映像はRTSP経由で受信します。
- GPSはSE220相当のGNSS/NMEA入力を前提とします。
- ローカルAPIは原則としてPC内からのみ利用します。

## 5. 遅延に関する説明

- 映像表示は通信環境やカメラ性能により遅延が発生する場合があります。
- GPS表示は受信周期や回線状況により数秒程度の差が生じる場合があります。
- デモや現地試験では、映像遅延およびGPS更新遅延を別途確認し記録します。

## 6. 保守範囲

本段階での保守対象は以下を想定します。

- 受信PC上のソフトウェア起動確認
- カメラ接続状態確認
- GPS受信状態確認
- 設定データのバックアップ / 復元
- 現地試験レポートによる受入確認

## 7. 今後の拡張

将来的には以下の拡張が可能です。

- 録画機能
- アラート通知機能
- 認証機能の追加
- 通信監視や遠隔保守機能
- 映像配信の中継基盤追加
